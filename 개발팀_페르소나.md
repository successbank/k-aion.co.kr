```
2. 40~60대가 3초 안에 이해하는 직관적 UI
3. 창업 상담 전환율을 극대화하는 CTA 설계
4. 모든 디바이스에서 완벽한 반응형 경험
```

|-----------|-----------|

| 정보 구조 | 사이트맵, 네비게이션 설계 |
| 와이어프레임 | 전 페이지 로우파이 → 미드파이 |
| 사용성 테스트 | 프로토타입 테스트, 개선점 도출 |

### UX 리서치 접근법
> "40~60대 사용자는 디지털 리터러시 수준이 다양합니다. 가장 낮은 수준의 사용자도 막힘없이 사용할 수 있어야 진짜 좋은 UX입니다. 추측이 아닌 데이터로 증명하겠습니다."

### 핵심 산출물
```
• 사용자 리서치 리포트
• 페르소나 문서 (3개 유형)
• 사용자 저니맵 (창업/소비자)
• 사이트맵 및 IA 문서
• 전 페이지 와이어프레임 (20+ 화면)
• 사용성 테스트 리포트
```

### 리서치 계획
| 단계 | 활동 | 기간 |
|------|------|------|
| Discovery | 이해관계자 인터뷰, 경쟁사 분석 | 1주차 |
| Research | 타겟 사용자 인터뷰 (8~10명) | 1~2주차 |
| Define | 페르소나, 저니맵, IA 설계 | 2주차 |
| Wireframe | 로우파이 → 미드파이 | 2~3주차 |
| Test | 프로토타입 사용성 테스트 | 4주차 |

### 디자인 원칙
> "애니메이션은 장식이 아니라 커뮤니케이션입니다. 사용자에게 '지금 무슨 일이 일어나고 있는지'를 0.3초 안에 알려줘야 합니다. 단, 40~60대에게는 과하지 않게, 자연스럽게."

### 핵심 산출물
```
• 인터랙션 가이드라인 문서
• 버튼/폼 마이크로 인터랙션 (20+ 패턴)
• 메인 페이지 스크롤 애니메이션 시퀀스
• Lottie 애니메이션 파일 (15+ 파일)
• 개발팀 인수인계용 애니메이션 스펙 문서
```

### 애니메이션 성능 가이드
| 항목 | 기준 |
|------|------|
| Duration | 150ms ~ 400ms (기본 300ms) |
| Easing | ease-out (진입), ease-in-out (전환) |
| FPS | 60fps 유지 |
| 접근성 | prefers-reduced-motion 대응 필수 |

```sql
  phone_encrypted TEXT NOT NULL,  -- AES-256 암호화
  preferred_region VARCHAR(100),
  message TEXT,
  status consultation_status DEFAULT 'pending',
  privacy_agreed_at TIMESTAMP NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  contacted_at TIMESTAMP,
  completed_at TIMESTAMP
);

  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(200) NOT NULL,
  address TEXT NOT NULL,
  phone VARCHAR(20),
  latitude DECIMAL(10, 8),
  longitude DECIMAL(11, 8),
  operating_hours JSONB,
  region_code VARCHAR(10),  -- 시도 코드
  district_code VARCHAR(10),  -- 구군 코드
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_stores_region ON stores(region_code, district_code);
CREATE INDEX idx_stores_location ON stores USING GIST (
  ll_to_earth(latitude, longitude)
```

| FID | ≤ 100ms | 100ms ~ 300ms | > 300ms |
| CLS | ≤ 0.1 | 0.1 ~ 0.25 | > 0.25 |
| TTFB | ≤ 600ms | 600ms ~ 1s | > 1s |

### 부하 테스트 시나리오
```javascript
// k6 부하 테스트 스크립트
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '2m', target: 100 },   // 2분간 100명까지 증가
    { duration: '5m', target: 100 },   // 5분간 100명 유지
    { duration: '2m', target: 200 },   // 2분간 200명까지 증가
    { duration: '5m', target: 200 },   // 5분간 200명 유지
    { duration: '2m', target: 0 },     // 2분간 0명으로 감소
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'],  // 95%가 500ms 이내
    http_req_failed: ['rate<0.01'],    // 실패율 1% 미만
  },
};

export default function () {
  const res = http.get('https://health1zone.co.kr/');
  check(res, {
    'status is 200': (r) => r.status === 200,
    'response time < 500ms': (r) => r.timings.duration < 500,
  });
  sleep(1);
}
```
